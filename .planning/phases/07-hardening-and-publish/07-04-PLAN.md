```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

# Required for provenance on release workflow (not on CI, but documenting locally
# that this workflow NEVER publishes — only the release.yml workflow does).
permissions:
  contents: read

jobs:
  # Primary build + quality-gates job.
  # D-03: keep existing attw + publint on the Node leg, do not restructure into a separate workflow.
  # D-22: size + tarball-audit + tree-shake run on every PR in this job.
  build:
    name: Build + gates (node 22)
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: 10

      - uses: actions/setup-node@v4
        with:
          node-version: '22'
          cache: 'pnpm'

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Build library
        run: pnpm build

      - name: Run unit tests
        run: pnpm test

      - name: Lint + format check
        run: pnpm check

      - name: attw — type resolution check
        run: cd packages/whisper && npx attw --pack --profile node16

      - name: publint — package export lint
        run: cd packages/whisper && npx publint .

      - name: Size budgets (D-19)
        run: pnpm size

      - name: Tarball content audit (D-21)
        run: pnpm tarball:audit

      # Pack the library so the smoke matrix can install the exact artifact CI just built.
      # This guarantees all 4 runtimes test the identical tarball bits.
      # Note: this `pnpm pack` re-runs the prepack hook (strips source maps) so the uploaded
      # artifact is what real consumers would receive.
      - name: Pack library for smoke matrix
        run: |
          cd packages/whisper
          pnpm pack --pack-destination "$GITHUB_WORKSPACE/e2e/smoke"

      # Populate e2e/smoke/node_modules so the tree-shake check can resolve @wardbox/whisper
      # via esbuild. Mirrors what `scripts/smoke/run.mjs` does locally.
      - name: Install smoke fixture
        working-directory: e2e/smoke
        run: pnpm install --force --ignore-workspace

      - name: Tree-shake verification (D-20)
        run: node e2e/smoke/tree-shake/check.mjs

      - name: Upload packed tarball for smoke matrix
        uses: actions/upload-artifact@v4
        with:
          name: whisper-tarball
          path: e2e/smoke/wardbox-whisper-*.tgz
          retention-days: 1
          if-no-files-found: error

  # Runtime smoke matrix.
  # D-01: 4 legs — node-esm, node-cjs, deno, bun.
  # D-02: all legs on ubuntu-latest only.
  # D-07: no RIOT_API_KEY, no live API calls.
  # D-08: Deno uses nodeModulesDir auto workaround, Bun uses bun install.
  smoke:
    name: Smoke (${{ matrix.runtime }})
    needs: build
    runs-on: ubuntu-latest
    strategy:
      fail-fast: false
      matrix:
        runtime: [node-esm, node-cjs, deno, bun]
    steps:
      - uses: actions/checkout@v4

      - name: Download packed tarball
        uses: actions/download-artifact@v4
        with:
          name: whisper-tarball
          path: e2e/smoke/

      # Node legs share pnpm + Node setup
      - name: Setup pnpm
        if: startsWith(matrix.runtime, 'node')
        uses: pnpm/action-setup@v4
        with:
          version: 10

      - name: Setup Node
        if: startsWith(matrix.runtime, 'node')
        uses: actions/setup-node@v4
        with:
          node-version: '22'
          cache: 'pnpm'

      - name: Setup Deno
        if: matrix.runtime == 'deno'
        uses: denoland/setup-deno@v2.0.4
        with:
          deno-version: v2.x

      - name: Setup Bun
        if: matrix.runtime == 'bun'
        uses: oven-sh/setup-bun@v2.2.0
        with:
          bun-version: latest

      # node-esm leg: pnpm install --force (busts tarball cache) + typecheck + run smoke.mjs
      - name: Run Node ESM smoke
        if: matrix.runtime == 'node-esm'
        working-directory: e2e/smoke
        run: |
          pnpm install --force --ignore-workspace
          pnpm typecheck
          node smoke.mjs

      # node-cjs leg: same install, run smoke.cjs (exercises the 'require' condition)
      - name: Run Node CJS smoke
        if: matrix.runtime == 'node-cjs'
        working-directory: e2e/smoke
        run: |
          pnpm install --force --ignore-workspace
          node smoke.cjs

      # deno leg: populate node_modules via pnpm, then run with nodeModulesDir=auto
      # (workaround path locked by user decision — no npm: registry lookup, no bootstrap publish)
      - name: Run Deno smoke
        if: matrix.runtime == 'deno'
        working-directory: e2e/smoke
        run: |
          # Deno doesn't come with pnpm — install pnpm separately for this leg to populate node_modules
          npm install -g pnpm@10
          pnpm install --force --ignore-workspace
          deno run -A --node-modules-dir=auto smoke_deno.ts

      # bun leg: bun install --force + bun run smoke_bun.ts
      - name: Run Bun smoke
        if: matrix.runtime == 'bun'
        working-directory: e2e/smoke
        run: |
          bun install --force
          bun run smoke_bun.ts
```

KEY POINTS:
- D-03 preserved: `build`, `test`, `check`, `attw`, `publint` all still run on the Node leg in the same order as the original ci.yml.
- D-22 new gates: `pnpm size`, `pnpm tarball:audit`, and `node e2e/smoke/tree-shake/check.mjs` run on EVERY PR in the primary `build` job. These catch regressions at PR time, not release time.
- Tree-shake check runs in the `build` job (not the matrix) because it only needs to run once — it asserts library properties, not runtime behavior.
- Artifact upload: the packed tarball from the primary job is downloaded by every matrix leg, guaranteeing all 4 runtimes test the same bits.
- Matrix uses `fail-fast: false` so a Bun-specific bug doesn't mask a Deno-specific bug.
- Deno leg has a small extra step: it runs `npm install -g pnpm@10` first because `denoland/setup-deno@v2.0.4` doesn't pre-install pnpm and we still need pnpm to populate the smoke fixture's node_modules. This is ~5 seconds extra and keeps the workaround path intact.
- Action versions pinned to explicit tags per RESEARCH supply-chain guidance: `denoland/setup-deno@v2.0.4`, `oven-sh/setup-bun@v2.2.0`, `actions/upload-artifact@v4`, `actions/download-artifact@v4`. Existing `actions/checkout@v4`, `pnpm/action-setup@v4`, `actions/setup-node@v4` unchanged per D-03 "do not restructure."
- `permissions: contents: read` at workflow level explicitly documents that ci.yml NEVER writes — that's the release.yml's job (Plan 05).
- `if-no-files-found: error` on the artifact upload means a silent pack failure surfaces immediately.

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PR → CI | Untrusted PR code runs in the workflow; malicious PRs could attempt to exfiltrate secrets via `echo $NPM_TOKEN` or similar. |
| CI job → artifact | The packed tarball is uploaded to GitHub's artifact store and downloaded by the matrix. If tampered, the matrix would test different bits than the primary job built. |
| GitHub Action → third-party | `denoland/setup-deno@v2.0.4`, `oven-sh/setup-bun@v2.2.0`, `pnpm/action-setup@v4`, etc. all run untrusted code from third-party repos. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-19 | Spoofing | Workflow injection — PR author controls a file that's passed to shell script unquoted | mitigate | Every `run:` block uses literal strings and pnpm script invocations, no `${{ github.* }}` interpolation into shell commands (only in `if:` conditions), no user-controlled input flows into exec |
| T-07-20 | Tampering | Artifact substitution between build and smoke jobs | mitigate | `actions/upload-artifact@v4` / `download-artifact@v4` use GitHub's internal artifact store with workflow-run scoping; cross-workflow artifact access is not possible by default |
| T-07-21 | Information Disclosure | ci.yml accidentally uses a secret (NPM_TOKEN, RIOT_API_KEY) and echoes it | mitigate | `permissions: contents: read` at workflow level; no `secrets.*` references anywhere in the file; grep assertion in acceptance criteria blocks `RIOT_API_KEY` and NPM_TOKEN from appearing |
| T-07-22 | Denial of Service | Third-party action compromised and injects malicious code into every run | mitigate | All third-party actions pinned to explicit version tags (v2.0.4, v2.2.0, v4) — supply-chain guidance from RESEARCH. Future hardening: pin to SHAs. Accepted risk for Phase 7 per 2026 guidance. |
| T-07-23 | Elevation of Privilege | ci.yml grants write permission somewhere and a compromised action exploits it | mitigate | `permissions: contents: read` at workflow root — zero write permissions. Any future step needing write must be explicitly granted. |
| T-07-24 | Tampering | schema-drift.yml inadvertently modified by this plan, breaking the RIOT_API_KEY flow | mitigate | Task 1 acceptance criterion runs `git diff --exit-code .github/workflows/schema-drift.yml` — zero-tolerance unchanged check |
